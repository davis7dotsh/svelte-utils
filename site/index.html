svelte-utils

Test-drive self-contained .svelte components: instant playground previews, svelte-check, and the official Svelte autofixer — from one tiny CLI, against a core server on any machine on your network.

⚠️ Not affiliated with the Svelte team. This is an independent open-source project that reuses the MIT-licensed playground from sveltejs/svelte.dev and the @sveltejs/mcp autofixer. All credit for those goes to the Svelte contributors.

Install the CLI

curl -fsSL https://davis7dotsh.github.io/svelte-utils/install.sh | sh

One dependency-free Node script. Requires Node 20+.

Set up a core (server machine)

git clone https://github.com/davis7dotsh/svelte-utils
cd svelte-utils && pnpm install
svelte-utils daemon install   # systemd service on 0.0.0.0:5175, starts at boot

Use it from anywhere

svelte-utils config set host http://your-server:5175

svelte-utils open Component.svelte            # live playground preview
svelte-utils check Component.svelte           # svelte-check diagnostics
svelte-utils best-practices Component.svelte  # Svelte autofixer

Re-running open after an edit hot-updates the already-open browser tab. --json flags make check and best-practices pipeline-friendly.
